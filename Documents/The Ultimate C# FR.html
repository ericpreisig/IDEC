Les bases du C#

Pour écrire un code efficace en C#, il est important d'en comprendre d'abord les bases. Dans cette section de la fiche récapitulative ultime C#, nous couvrirons les concepts fondamentaux du langage, y compris les types de données, les variables, les opérateurs et les structures de contrôle.

Variables et Types de Données

En C#, les variables sont utilisées pour stocker des valeurs de données qui peuvent être utilisées et manipulées tout au long de votre programme. Il existe plusieurs types de données différents que vous pouvez utiliser en C#, notamment :

  • int - utilisé pour stocker des nombres entiers.
  • float - utilisé pour stocker des nombres à virgule flottante.
  • double - utilisé pour stocker des nombres à virgule flottante plus grands/plus précis.
  • bool - utilisé pour stocker des valeurs vrai/faux (true/false).
  • string - utilisé pour stocker des valeurs textuelles.

Voici un exemple de déclaration et d'utilisation de variables en C# :

int myNumber = 10;
float myFloat = 3.14f;
double myDouble = 1.23456789;
bool myBool = true;
string myString = "Hello, world! This is Tech with Christian :)";

Opérateurs

Les opérateurs sont des symboles utilisés pour effectuer des opérations mathématiques ou logiques sur des données dans un programme. Le C# inclut une variété d'opérateurs, notamment les opérateurs arithmétiques (+, -, *, /), les opérateurs de comparaison (>, <, ==) et les opérateurs logiques (&&, ||, !). Vous trouverez ci-dessous quelques exemples pour référence.

Opérateurs Arithmétiques

Les opérateurs arithmétiques sont utilisés pour effectuer des opérations mathématiques telles que l'addition, la soustraction, la multiplication et la division. Voici quelques exemples d'opérateurs arithmétiques en C# :

int a = 10;
int b = 5;

int sum = a + b;    // addition
int difference = a - b;    // subtraction
int product = a * b;    // multiplication
int quotient = a / b;    // division
int remainder = a % b;    // modulus (returns the remainder of a division operation)

Opérateurs de Comparaison

Les opérateurs de comparaison sont utilisés pour comparer deux valeurs et retourner une valeur booléenne ( true ou false ) en fonction de la comparaison. Voici quelques exemples d'opérateurs de comparaison en C# :

int a = 10;
int b = 5;

bool isEqual = a == b;    // equal to
bool isNotEqual = a != b;    // not equal to
bool isGreater = a > b;    // greater than
bool isLess = a < b;    // less than
bool isGreaterOrEqual = a >= b;    // greater than or equal to
bool isLessOrEqual = a <= b;    // less than or equal to

Opérateurs Logiques

Les opérateurs logiques sont utilisés pour effectuer des opérations logiques sur des valeurs booléennes. Voici quelques exemples d'opérateurs logiques en C# :

bool a = true;
bool b = false;

bool andResult = a && b;    // logical AND
bool orResult = a || b;    // logical OR
bool notResult = !a;    // logical NOT

Opérateurs d'Assignation

Les opérateurs d'assignation sont utilisés pour attribuer une valeur à une variable. Voici quelques exemples d'opérateurs d'assignation en C# :

int a = 10;
int b = 5;

a += b;    // equivalent to a = a + b
a -= b;    // equivalent to a = a - b
a *= b;    // equivalent to a = a * b
a /= b;    // equivalent to a = a / b
a %= b;    // equivalent to a = a % b

Ce ne sont là que quelques exemples des nombreux opérateurs disponibles en C#. En comprenant et en maîtrisant ces opérateurs, vous pourrez effectuer une grande variété d'opérations sur des variables et des valeurs dans vos programmes C#.

Structures de contrôle

Les structures de contrôle sont utilisées pour contrôler le flux d'exécution dans un programme. En C#, il existe plusieurs structures de contrôle, notamment les instructions if-else, les instructions switch et les boucles ( for, while, et do-while ).

int myNumber = 10;

if (myNumber < 5)
{
    Console.WriteLine("The number is less than 5");
}
else if (myNumber > 15)
{
    Console.WriteLine("The number is greater than 15");
}
else
{
    Console.WriteLine("The number is between 5 and 15");
}

Une autre structure de contrôle courante est la boucle for, qui vous permet d'itérer à travers un ensemble de valeurs un certain nombre de fois. Voici un exemple avec une boucle for :

for (int i = 0; i < 10; i++)
{
    Console.WriteLine("The value of i is: " + i);
}

Fonctions

Les fonctions sont des blocs de code qui peuvent être appelés depuis d'autres parties de votre programme. Elles sont utiles pour organiser votre code et le rendre plus réutilisable. Voici un exemple d'une fonction qui prend deux paramètres entiers et renvoie leur somme :

int AddNumbers(int a, int b)
{
    return a + b;
}

Vous pouvez appeler cette fonction à partir d'autres parties de votre code comme ceci :

int result = AddNumbers(10, 5);
Console.WriteLine("The result is: " + result);

Comprendre ces concepts de base est essentiel pour écrire un code C# efficace. En maîtrisant les bases, vous serez en mesure d'écrire des programmes plus complexes avec confiance.

Programmation Orientée Objet en C#

La programmation orientée objet (POO) est un paradigme de programmation qui met l'accent sur l'utilisation d'objets, de classes et de l'héritage pour organiser et structurer le code. Le C# est un langage orienté objet et prend en charge toutes les fonctionnalités de la POO. Dans cette section de la fiche récapitulative ultime C#, nous couvrirons les concepts clés de la POO en C#, avec quelques exemples de code.

Classes et Objets

En C#, une classe est un plan pour créer des objets. Une classe définit les propriétés et les méthodes d'un objet, et un objet est une instance d'une classe. Les objets sont utilisés pour stocker des données et effectuer des opérations sur ces données.

class Person
{
    public string Name { get; set; }
    public int Age { get; set; }
    
    public void SayHello()
    {
        Console.WriteLine($"Hello, my name is {Name} and I'm {Age} years old.");
    }
}

Person john = new Person();
john.Name = "John";
john.Age = 30;
john.SayHello(); // Output: Hello, my name is John and I'm 30 years old.

Héritage

L'héritage est une fonctionnalité clé de la POO (Programmation Orientée Objet) qui permet à une classe d'hériter des propriétés et des méthodes d'une autre classe. En C#, une classe peut hériter d'une seule classe de base mais peut implémenter plusieurs interfaces.

class Animal
{
    public string Name { get; set; }
    
    public void Eat()
    {
        Console.WriteLine($"{Name} is eating.");
    }
}

class Dog : Animal
{
    public void Bark()
    {
        Console.WriteLine($"{Name} is barking.");
    }
}

Dog fido = new Dog();
fido.Name = "Fido";
fido.Eat(); // Output: Fido is eating.
fido.Bark(); // Output: Fido is barking.

Polymorphisme

Le polymorphisme est la capacité d'un objet à prendre plusieurs formes. En C#, le polymorphisme peut être atteint par le remplacement de méthode (method overriding), où une sous-classe fournit sa propre implémentation d'une méthode héritée de sa classe parente.

class Shape
{
    public virtual void Draw()
    {
        Console.WriteLine("Drawing a shape.");
    }
}

class Circle : Shape
{
    public override void Draw()
    {
        Console.WriteLine("Drawing a circle.");
    }
}

class Square : Shape
{
    public override void Draw()
    {
        Console.WriteLine("Drawing a square.");
    }
}

Shape shape1 = new Circle();
shape1.Draw(); // Output: Drawing a circle.

Shape shape2 = new Square();
shape2.Draw(); // Output: Drawing a square.

Abstraction

L'abstraction est le processus qui consiste à masquer les détails inutiles tout en ne montrant que les caractéristiques essentielles d'un objet. En C#, l'abstraction peut être réalisée par le biais de classes abstraites et d'interfaces.

abstract class Animal
{
    public abstract string Name { get; set; }
    public abstract void MakeSound();
}

class Dog : Animal
{
    public override string Name { get; set; }

    public override void MakeSound()
    {
        Console.WriteLine("Woof!");
    }
}

Animal dog = new Dog();
dog.Name = "Fido";
dog.MakeSound(); // Output: Woof!

Encapsulation

L'encapsulation est la pratique consistant à restreindre l'accès à certains composants d'un objet pour empêcher les modifications indésirables. En C#, l'encapsulation peut être réalisée par le biais de modificateurs d'accès tels que public, private, et protected.

class BankAccount
{
    private double balance;
    
    public void Deposit(double amount)
    {
        balance += amount;
    }

    public void Withdraw(double amount)
    {
        if (balance >= amount)
        {
            balance -= amount;
        }
    }
    
    public double GetBalance()
    {
        return balance;
    }
}

BankAccount account = new BankAccount();
account.Deposit(1000);
account.Withdraw(500);
double balance = account.GetBalance(); // balance is 500

Collections en C#

Les collections en C# sont utilisées pour stocker des groupes d'objets liés. Il existe plusieurs types de collections disponibles en C#, notamment les tableaux, les listes, les dictionnaires et les ensembles.

Tableaux (Arrays)

Un array est une collection d'éléments du même type. La longueur d'un tableau est fixe et ne peut pas être modifiée une fois qu'il est créé.

int[] numbers = new int[3] { 1, 2, 3 };
Console.WriteLine(numbers[0]); // Output: 1
Console.WriteLine(numbers[1]); // Output: 2
Console.WriteLine(numbers[2]); // Output: 3

Listes (Lists)

Une liste est une collection dynamique d'éléments du même type. Contrairement aux tableaux, la longueur d'une liste peut être modifiée après sa création.

List<string> names = new List<string>();
names.Add("Alice");
names.Add("Bob");
names.Add("Charlie");
Console.WriteLine(names[1]); // Output: Bob
names.Remove("Charlie");
Console.WriteLine(names.Count); // Output: 2

Dictionnaires (Dictionaries)

Un dictionnaire est une collection de paires clé-valeur. Chaque clé doit être unique, et la valeur associée à une clé peut être de n'importe quel type.

Dictionary<string, int> ages = new Dictionary<string, int>();
ages.Add("Alice", 25);
ages.Add("Bob", 30);
ages.Add("Charlie", 35);
Console.WriteLine(ages["Bob"]); // Output: 30
ages.Remove("Charlie");
Console.WriteLine(ages.Count); // Output: 2

Ensembles (Sets)

Un ensemble est une collection d'éléments uniques. En C#, les ensembles sont implémentés en utilisant la classe HashSet.

HashSet<int> numbers = new HashSet<int>();
numbers.Add(1);
numbers.Add(2);
numbers.Add(3);
numbers.Add(3); // Adding a duplicate value has no effect
Console.WriteLine(numbers.Count); // Output: 3
numbers.Remove(2);
Console.WriteLine(numbers.Count); // Output: 2

J'ai utilisé des HashSets pour m'assurer d'obtenir une liste d'éléments uniques sans aucun doublon. Disons que vous avez une grande collection de données et que vous souhaitez en supprimer les doublons. Une façon de faire est d'itérer à travers la collection et d'ajouter chaque élément à un HashSet. Puisqu'un HashSet ne peut contenir que des valeurs uniques, tous les doublons seront automatiquement supprimés.

using System;
using System.Collections.Generic;

class Program
{
    static void Main(string[] args)
    {
        // Create a list with some duplicate values
        List<int> numbers = new List<int> { 1, 2, 3, 4, 5, 2, 3, 6, 7, 1 };

        // Create a HashSet and add all the numbers to it
        HashSet<int> uniqueNumbers = new HashSet<int>();
        foreach (int number in numbers)
        {
            uniqueNumbers.Add(number);
        }

        // Print out the unique numbers
        foreach (int number in uniqueNumbers)
        {
            Console.WriteLine(number);
        }
    }
}

Dans cet exemple, je crée une liste d'entiers avec quelques valeurs en double. Je crée ensuite un HashSet et j'y ajoute tous les nombres en utilisant une boucle foreach. Puisque le HashSet ne peut contenir que des valeurs uniques, tous les doublons sont automatiquement supprimés. Enfin, nous itérons sur le HashSet et imprimons les nombres uniques.

Gestion des exceptions en C#

En C#, les exceptions sont utilisées pour gérer les erreurs d'exécution qui surviennent pendant l'exécution d'un programme. La gestion des exceptions permet aux développeurs de gérer les erreurs de manière élégante et d'éviter que le programme ne plante.

Blocs Try-Catch

Un bloc try-catch est utilisé pour gérer les exceptions en C#. Le code susceptible de générer une exception est placé dans le bloc try, et toutes les exceptions qui sont levées sont capturées et gérées par le bloc catch.

try
{
    int x = 0;
    int y = 10 / x; // This will throw a DivideByZeroException
}
catch (DivideByZeroException e)
{
    Console.WriteLine("Error: " + e.Message);
}

Bloc Finally

Un bloc finally peut être utilisé pour exécuter du code qui doit toujours être exécuté, qu'une exception soit levée ou non.

try
{
    // Code that might throw an exception
}
catch (Exception e)
{
    Console.WriteLine("Error: " + e.Message);
}
finally
{
    // Code that should always be run
}

Lancer des exceptions (Throwing Exceptions)

En C#, les développeurs peuvent également lever explicitement des exceptions à l'aide du mot-clé throw. Cela peut être utile lorsqu'une gestion d'erreur personnalisée est nécessaire.

try
{
    string input = Console.ReadLine();
    if (string.IsNullOrEmpty(input))
    {
        throw new Exception("Input cannot be empty.");
    }
}
catch (Exception e)
{
    Console.WriteLine("Error: " + e.Message);
}

Exceptions personnalisées

Les développeurs peuvent également créer des classes d'exceptions personnalisées pour gérer des types d'erreurs spécifiques.

public class CustomException : Exception
{
    public CustomException(string message) : base(message)
    {
    }
}

try
{
    int x = 0;
    int y = 10 / x;
    throw new CustomException("Custom exception message.");
}
catch (CustomException e)
{
    Console.WriteLine("Error: " + e.Message);
}

Entrées et Sorties de fichiers en C#

Le C# fournit un ensemble de classes et de méthodes qui peuvent être utilisées pour lire et écrire dans des fichiers. Cela peut être utile lorsque vous travaillez avec des données qui doivent être conservées au-delà de la durée de vie du programme.

Lire depuis un fichier

La classe StreamReader peut être utilisée pour lire des données à partir d'un fichier. Le code ci-dessous montre comment lire toutes les lignes d'un fichier texte en utilisant StreamReader :

using (StreamReader reader = new StreamReader("file.txt"))
{
    string line;
    while ((line = reader.ReadLine()) != null)
    {
        Console.WriteLine(line);
    }
}

Écrire dans un fichier

La classe StreamWriter peut être utilisée pour écrire des données dans un fichier. Le code ci-dessous montre comment écrire une chaîne de caractères dans un fichier texte en utilisant StreamWriter :

using (StreamWriter writer = new StreamWriter("file.txt"))
{
    writer.WriteLine("Hello, world!");
}

Ajouter à un fichier

Pour ajouter du contenu à un fichier existant au lieu de l'écraser, passez true comme deuxième argument au constructeur StreamWriter :

using (StreamWriter writer = new StreamWriter("file.txt", true))
{
    writer.WriteLine("This line will be appended to the file.");
}

Fichiers Binaires

Les fichiers binaires contiennent des données qui ne sont pas sous une forme lisible par l'homme, comme des images ou des fichiers audio. Les classes BinaryReader et BinaryWriter peuvent être utilisées pour lire et écrire des données binaires dans un fichier. Le code ci-dessous montre comment écrire des données binaires dans un fichier :

using (BinaryWriter writer = new BinaryWriter(File.Open("file.bin", FileMode.Create)))
{
    byte[] data = { 0x00, 0x01, 0x02, 0x03 };
    writer.Write(data);
}

Pour lire des données binaires à partir d'un fichier, utilisez la classe BinaryReader :

using (BinaryReader reader = new BinaryReader(File.Open("file.bin", FileMode.Open)))
{
    byte[] data = reader.ReadBytes(4);
    Console.WriteLine(BitConverter.ToString(data));
}

LINQ en C#

Language-Integrated Query (LINQ) est une fonctionnalité puissante de C# qui permet aux développeurs d'interroger des données à partir de diverses sources en utilisant une syntaxe commune. LINQ peut être utilisé avec des objets, des tableaux, des collections et des bases de données.

Syntaxe LINQ

La syntaxe de base d'une requête LINQ se compose de trois parties : la source de données, l'opération de requête et le résultat.

var result = from item in source
             where condition
             select item;

Filtrer les données

Dans cet exemple, nous allons utiliser LINQ pour filtrer une liste d'entiers.

List<int> numbers = new List<int> { 1, 2, 3, 4, 5 };

var result = from num in numbers
             where num % 2 == 0
             select num;

foreach (int num in result)
{
    Console.WriteLine(num);
}

Cela donnera le résultat suivant :

2
4

Projeter les données

Dans cet exemple, nous allons utiliser LINQ pour projeter une liste de chaînes de caractères en majuscules.

List<string> words = new List<string> { "hello", "world" };

var result = from word in words
             select word.ToUpper();

foreach (string word in result)
{
    Console.WriteLine(word);
}

Cela donnera le résultat suivant :

HELLO
WORLD

Joindre les données

Dans cet exemple, nous allons utiliser LINQ pour joindre deux listes d'objets.

class Person
{
    public string Name { get; set; }
    public int Age { get; set; }
}

List<Person> people = new List<Person>
{
    new Person { Name = "Alice", Age = 25 },
    new Person { Name = "Bob", Age = 30 },
    new Person { Name = "Charlie", Age = 35 }
};

List<string> cities = new List<string> { "Amsterdam", "Berlin", "Chicago" };

var result = from person in people
             join city in cities on person.Name[0] equals city[0]
             select new { Name = person.Name, City = city };

foreach (var item in result)
{
    Console.WriteLine(item.Name + " - " + item.City);
}

Cela donnera le résultat suivant :

Alice - Amsterdam
Bob - Berlin
Charlie - Chicago

Multithreading en C#

Le multithreading est le processus d'exécution de plusieurs threads simultanément dans un seul processus. Les threads permettent à un programme d'effectuer plusieurs tâches en même temps, améliorant ainsi les performances et la réactivité.

Bases des Threads

Un thread est un chemin d'exécution au sein d'un programme. Chaque processus a au moins un thread, le thread principal, qui est créé automatiquement au démarrage du processus. Des threads supplémentaires peuvent être créés pour exécuter des tâches en parallèle avec le thread principal.

En d'autres termes. Les threads sont utilisés pour effectuer plusieurs tâches simultanément dans un programme. En utilisant des threads, un programme peut atteindre la concurrence, ce qui signifie qu'il peut faire plusieurs choses en même temps. Cela peut conduire à de meilleures performances et à une meilleure réactivité du programme, en particulier dans les cas où le programme effectue de nombreuses opérations bloquantes ou chronophages. Voici deux exemples pour vous :

  1. Vous créez une application de partage de fichiers qui permet aux utilisateurs d'uploader et de télécharger des fichiers. Lorsqu'un utilisateur télécharge un fichier volumineux, l'application peut prendre beaucoup de temps à le traiter. Sans utiliser de threads, l'application peut devenir inactive pour les autres utilisateurs jusqu'à ce que le processus de téléchargement soit terminé. Cependant, si vous utilisez des threads pour gérer les téléchargements de fichiers, l'utilisateur peut continuer à utiliser l'application pendant que le téléchargement est traité en arrière-plan.
  2. Un serveur web qui doit gérer plusieurs requêtes simultanément. En utilisant des threads, le serveur web peut créer un nouveau thread pour chaque requête entrante et les gérer simultanément, ce qui améliore les performances globales du serveur.

Créer et démarrer des threads

En C#, vous pouvez créer et démarrer un nouveau thread en utilisant la classe Thread.

void DoWork()
{
    // Perform some work
}

Thread thread = new Thread(new ThreadStart(DoWork));
thread.Start();

Cela créera un nouveau thread et commencera à exécuter la méthode DoWork en arrière-plan.

Synchronisation

Lorsque plusieurs threads accèdent à des ressources partagées, il y a un risque de corruption de données et de conditions de concurrence (race conditions). Pour éviter cela, des techniques de synchronisation peuvent être utilisées pour s'assurer qu'un seul thread peut accéder à une ressource partagée à la fois.

Une technique courante de synchronisation est l'utilisation de verrous (locks).

class Counter
{
    private int count = 0;
    private object lockObject = new object();

    public void Increment()
    {
        lock (lockObject)
        {
            count++;
        }
    }

    public int Count
    {
        get { return count; }
    }
}

Counter counter = new Counter();

Thread thread1 = new Thread(() =>
{
    for (int i = 0; i < 1000000; i++)
    {
        counter.Increment();
    }
});

Thread thread2 = new Thread(() =>
{
    for (int i = 0; i < 1000000; i++)
    {
        counter.Increment();
    }
});

thread1.Start();
thread2.Start();

thread1.Join();
thread2.Join();

Console.WriteLine(counter.Count);

Cela affichera 2000000, ce qui est le résultat attendu puisque les deux threads incrémentent le compteur un million de fois chacun.

Sécurité des threads

La sécurité des threads (thread safety) est la propriété d'un objet ou d'une méthode qui garantit qu'il peut être utilisé en toute sécurité par plusieurs threads en même temps sans causer de corruption de données ou de conditions de concurrence.

Une façon d'obtenir la sécurité des threads est d'utiliser le mot-clé lock, comme démontré dans l'exemple précédent. Une autre façon est d'utiliser des objets immuables, qui ne peuvent pas être modifiés une fois qu'ils sont créés.

class ImmutableCounter
{
    private readonly int count = 0;

    public ImmutableCounter(int count)
    {
        this.count = count;
    }

    public ImmutableCounter Increment()
    {
        return new ImmutableCounter(count + 1);
    }

    public int Count
    {
        get { return count; }
    }
}

ImmutableCounter counter = new ImmutableCounter(0);

Thread thread1 = new Thread(() =>
{
    for (int i = 0; i < 1000000; i++)
    {
        counter = counter.Increment();
    }
});

Thread thread2 = new Thread(() =>
{
    for (int i = 0; i < 1000000; i++)
    {
        counter = counter.Increment();
    }
});

thread1.Start();
thread2.Start();

thread1.Join();
thread2.Join();

Console.WriteLine(counter.Count);

Cela affichera 2000000, ce qui est le résultat attendu puisque les deux threads créent de nouvelles instances de l'objet ImmutableCounter et incrémentent le compte. Étant donné que l'objet est immuable, il n'y a aucun risque de corruption des données ou de conditions de concurrence.

L'utilisation de threads peut rendre un programme plus réactif, améliorer ses performances et lui permettre d'effectuer plusieurs tâches simultanément, ce qui peut être essentiel dans certains types d'applications.

Fonctionnalités avancées du C#

Le C# est un langage de programmation puissant et polyvalent qui comprend une variété de fonctionnalités avancées pour vous aider à écrire du code efficace, flexible et maintenable. Voici quelques-unes des fonctionnalités avancées les plus utiles de C#, avec des exemples de code pour illustrer leur fonctionnement.

Délégués (Delegates)

Un delegate est un type qui représente une signature de méthode, vous permettant de passer des méthodes comme arguments à d'autres méthodes ou de les stocker comme variables. Les délégués sont couramment utilisés dans la gestion des événements et les scénarios de rappel (callbacks).

public delegate void MyDelegate(string message);

public class MyClass
{
    public event MyDelegate MyEvent;

    public void RaiseEvent(string message)
    {
        MyEvent?.Invoke(message);
    }
}

public class Program
{
    public static void Main()
    {
        MyClass myClass = new MyClass();
        myClass.MyEvent += MyEventHandler;
        myClass.RaiseEvent("Hello, world!");
    }

    public static void MyEventHandler(string message)
    {
        Console.WriteLine(message);
    }
}

Ce code crée un type délégué personnalisé MyDelegate, puis définit une classe MyClass avec un événement de ce type. Dans la méthode Main, nous créons une instance de MyClass, attachons une méthode MyEventHandler à l'événement, puis déclenchons l'événement avec un message. La méthode MyEventHandler est appelée et imprime le message dans la console.

Méthodes d'extension

Les méthodes d'extension vous permettent d'ajouter des méthodes à une classe existante sans modifier son code source. Ceci est utile lorsque vous souhaitez ajouter des fonctionnalités à une classe sur laquelle vous n'avez aucun contrôle, ou lorsque vous souhaitez regrouper des fonctionnalités associées dans une seule classe.

public static class StringExtensions
{
    public static bool IsPalindrome(this string str)
    {
        for (int i = 0; i < str.Length / 2; i++)
        {
            if (str[i] != str[str.Length - i - 1])
            {
                return false;
            }
        }
        return true;
    }
}

public class Program
{
    public static void Main()
    {
        string str1 = "racecar";
        string str2 = "hello";

        Console.WriteLine(str1.IsPalindrome());
        Console.WriteLine(str2.IsPalindrome());
    }
}

Ce code définit une classe statique StringExtensions avec une méthode d'extension IsPalindrome, qui détermine si une chaîne est un palindrome. Nous appelons ensuite cette méthode sur deux chaînes dans la méthode Main et affichons les résultats dans la console.

🤓
Un palindrome est un mot, une phrase ou un chiffre qui se lit de la même façon à l'endroit qu'à l'envers. Par exemple, "racecar" est un palindrome car si vous l'épelez à l'envers, il est toujours écrit "racecar".

3. Expressions Lambda

Les expressions lambda sont une manière concise de définir des méthodes anonymes, vous permettant d'écrire du code en ligne sans créer de méthode séparée.

public class Program
{
    public static void Main()
    {
        List<int> numbers = new List<int> { 1, 2, 3, 4, 5 };

        List<int> evenNumbers = numbers.Where(n => n % 2 == 0).ToList();

        foreach (int number in evenNumbers)
        {
            Console.WriteLine(number);
        }
    }
}

Ce code crée une liste d'entiers puis utilise une expression lambda pour filtrer les nombres pairs à l'aide de la méthode Where. Nous bouclons ensuite sur les nombres pairs et les affichons dans la console. Quand cela serait-il utile ?

Supposons que vous ayez une grande collection de nombres et que vous ayez besoin d'effectuer une opération uniquement sur les nombres pairs de la collection. Au lieu de parcourir manuellement la collection et de vérifier chaque nombre pour voir s'il est pair, vous pouvez utiliser LINQ pour filtrer la collection rapidement et efficacement.

Génériques

Les génériques vous permettent d'écrire du code qui peut fonctionner avec plusieurs types de données, ce qui améliore la réutilisation du code et réduit le besoin de conversion de type (casting).

public class Stack<T>
{
    private List<T> items = new List<T>();

    public void Push(T item)
    {
        items.Add(item);
    }

    public T Pop()
    {
        T item = items.Last();
        items.RemoveAt(items.Count - 1);
        return item;
    }
}

public class Program
{
    public static void Main()
    {
        Stack<int> stack = new Stack<int>();
        stack.Push(1);
        stack.Push(2);
        stack.Push(3);

        Console.WriteLine(stack.Pop());
        Console.WriteLine(stack.Pop());
        Console.WriteLine(stack.Pop());
    }
}

Ce code définit une classe Stack générique qui peut contenir n'importe quel type de données. Nous créons ensuite une instance de la classe avec int comme paramètre de type, empilons trois entiers sur la pile, puis les dépilons et les affichons dans la console. Cela pourrait être fait pour n'importe quel type.

Async/Await

Async / await est un modèle de programmation qui vous permet d'écrire un code asynchrone plus facile à lire et à maintenir que le code traditionnel basé sur les rappels (callbacks).

public class Program
{
    public static async Task Main()
    {
        HttpClient httpClient = new HttpClient();
        string response = await httpClient.GetStringAsync("https://jsonplaceholder.typicode.com/todos/1");
        Console.WriteLine(response);
    }
}

Ce code utilise la classe HttpClient pour effectuer une requête HTTP asynchrone vers une API web et récupérer des données. Nous utilisons le mot-clé async pour marquer la méthode Main comme asynchrone, puis nous utilisons le mot-clé await pour attendre le résultat de la requête HTTP avant de l'afficher dans la console. Voici le résultat de l'appel de l'API.

{
  "userId": 1,
  "id": 1,
  "title": "delectus aut autem",
  "completed": false
}

Attributs

Les attributs sont un moyen d'ajouter des métadonnées à votre code, vous permettant de spécifier des informations supplémentaires sur les types, méthodes et propriétés qui peuvent être utilisées par des outils et des bibliothèques.

[Serializable]
public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }
}

public class Program
{
    public static void Main()
    {
        Person person = new Person { Name = "Alice", Age = 30 };

        XmlSerializer serializer = new XmlSerializer(typeof(Person));
        StringWriter writer = new StringWriter();
        serializer.Serialize(writer, person);

        Console.WriteLine(writer.ToString());
    }
}

Ce code définit une classe Person avec deux propriétés et la marque avec l'attribut [Serializable]. Nous utilisons ensuite la classe XmlSerializer pour sérialiser une instance de la classe Person en une chaîne XML, que nous affichons dans la console.
